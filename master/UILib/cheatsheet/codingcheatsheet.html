Coding Cheat Sheet

Cidow Research Center

HTML

Guide

  • 모든 Markup은 W3C Validation Pass를 기본으로 합니다.
  • Markup의 모든 Element, Attribute는 소문자로 작성합니다.
  • Attribute의 Value 값은 큰 따옴표("") 안에 선언합니다.
  • 특수기호는 Entity Code로 변환하여 작성합니다.
  • 공백줄(빈행)은 사용하지 않습니다.
  • Code는 단계별로 1tab 들여쓰기합니다.
  • 1Tab의 크기는 4Space로 설정합니다.

CSS

Guide

  • CSS 파일은 link 태그를 사용한 External 방식을 사용합니다.
  • HTML과 같은 Encoding을 문서 첫 줄에 공백 없이 선언합니다.
  • 전역 선택자(*) 사용은 지양합니다.
  • 1Tab의 크기는 4Space로 설정합니다.
  • 선택자의 속성값이 하나일 경우 줄 바꿈 하지 않으며, 두 개 이상일 경우 줄 바꿈 및 들여쓰기합니다.
  • Property와 Value는 한 칸(1space)의 여백을 둡니다.
  • 다중 선택자 사용 시 세 번째 선택자 이후 줄 바꿈을 기본으로 합니다.
  • 콤마(,) 사용 시 콤마 뒤로 한 칸(1space)의 여백을 두며, 마지막 속성 뒤에도 세미콜론(;)을 작성합니다.

Javascript

Guide

  • 코딩단의 모든 Javascript 코드는 *.js 파일로 별도 저장하여 관리합니다.
  • 함수명과 변수명은 Camel 표기법을 사용합니다.
  • 전역변수로 사용되거나, 다른 변수와 구분이 필요하다고 판단되는 경우 변수명 앞에 _를 붙여줍니다.
  • 테스트 완료 후 console.log 와 같은 테스트 코드는 반드시 삭제합니다.
  • 구분을 위한 코멘트는 /* Comment */, 이해를 돕기 위한 코멘트는 // Comment 형식으로 작성합니다.

Library(jQuery)

  • 버전별 차이점을 확인하여, 해당 프로젝트에 맞는 버전의 파일을 사용 합니다.
  • 변수에 $('')를 할당할 경우 변수명 앞에 $를 붙여 줍니다.

Comment

Guide

  • 모든 주석은 영문으로 작성하는 것을 권장합니다.
  • HTML 문서의 정보를 위한 주석은 DTD 선언 다음에 작성합니다.
  • HTML 종료 주석은 시작 주석과 같게 작성하며 //로 구분합니다.
  • HTML/CSS 코드와 주석은 행으로 분리하며, 필요할 때 주석 내 줄 바꿈을 허용합니다.
  • 해당 영역에 할당된 id/class 혹은 function name 을 이용합니다.
HTML <-- HTML comment(begin) -->, <-- //HTML comment(end) -->
CSS /* CSS comment */
Javascript /* Js comment */, // Js inline comment

Naming

HTML/CSS

  • 의미, 상태의 순서로 작성 하며, 가급적 3단계를 넘지 않도록 합니다.
  • Id와 Class는 Camel 표기법을 사용하며, 레이아웃에는 가급적 id를 지정합니다.

Directory/File

  • 영문소문자만 사용합니다.
  • 형태_의미_상태의 순서로 네이밍 하며, 가급적 3단계를 넘지 않도록 합니다.
  • _(언더스코어)는 두 개 이상의 단어를 조합할 때만 사용합니다.
  • 단어와 숫자를 조합할 경우 _(언더스코어)를 사용하지 않습니다.
  • 뷰 페이지 문서 파일은 서비스 대분류에 따라 폴더를 생성하여 관리합니다.
  • 기타 파일은 해당 파일의 사용목적에 맞게 분리하여 폴더를 생성하여 관리합니다.
